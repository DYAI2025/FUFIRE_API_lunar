## Context

FuFirE's OpenAPI spec lives at `spec/openapi/openapi.json` (9000+
lines). It is regenerated from the live FastAPI app via
`scripts/export_openapi.py`. CI runs `--check` to fail-fast on drift.

CLAUDE.md mandates regenerating after **any** endpoint/model change,
including:

- Widening a `Literal[...]` in `bazi_engine/types.py` or any router.
- Adding/removing fields on a Pydantic request/response model.
- Adding a new endpoint or response_model.
- Changing `bazi_engine.__version__`.

The drift check is in `tests/test_openapi_contract.py` and the CI
job — but waiting for CI is slow. This skill does the local
roundtrip in seconds.

## Your Task

Regenerate the OpenAPI spec and verify the drift check is clean.

### Steps

1. **Regenerate:**
   ```bash
   source .venv/bin/activate 2>/dev/null || true
   python scripts/export_openapi.py
   ```
   Expected: `Written: …/spec/openapi/openapi.json`.

2. **Verify no drift:**
   ```bash
   python scripts/export_openapi.py --check
   ```
   Expected: `OK: OpenAPI spec is up-to-date.`
   If it still says `FAIL: OpenAPI spec drifted` after a fresh
   regenerate, something is non-deterministic in the app — stop and
   investigate; this is a real bug, not a routine drift.

3. **Show the diff:**
   ```bash
   git diff --stat spec/openapi/openapi.json
   git diff spec/openapi/openapi.json | head -60
   ```
   Report what changed at a high level (e.g. "TLST added to
   `TimeStandard` enum at 3 paths") so reviewers understand the
   commit.

4. **Run the contract test** (optional, costs ~1 s):
   ```bash
   python -m pytest tests/test_openapi_contract.py -q
   ```

### Guardrails

- Do NOT commit the spec yourself unless the user explicitly asks.
  Show the diff and let the user `git add` it as part of their
  feature commit.
- If `scripts/export_openapi.py` doesn't exist or fails, report the
  exact error — do not paper over with a hand-rolled regen.
- A persistent drift after re-regenerate usually means
  `bazi_engine/__init__.py::__version__` and `pyproject.toml` are
  out of sync (DEV-2026-003) — flag this if you see version-string
  drift in the diff.
- Do NOT touch `tests/test_openapi_contract.py` "to make it pass".
  If the contract test fails, the spec is the source of truth.
