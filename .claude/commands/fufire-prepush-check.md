## Context

The FuFirE working tree has four persistent stray paths that are
**not** part of any feature work and must never be pushed:

| Stray | Why it's here | Action |
|---|---|---|
| `/0` | zero-byte file at repo root, accidental | should be in `.gitignore` |
| `ruvector.db` | tool state (~1.5 MB SQLite-ish blob) | should be in `.gitignore` |
| `.swarm/` | claude-flow / swarm tool runtime state | should be in `.gitignore` |
| `.claude/homunculus/observations.jsonl` | session log for the homunculus observer | tracked despite `.gitignore` rule (`.claude/`) — pre-existing; **never** stage in feature commits |

`.gitignore` already has `.DS_Store`, `.venv/`, `__pycache__/`, etc.,

staging to avoid leaking them.

## Your Task

Before a `/ship` invocation: surface the strays, optionally fix
the `.gitignore`, and confirm the staging set is clean.

### Steps

1. **List current staging state:**
   ```bash
   git status --short
   ```

2. **Detect strays among the un-staged or untracked entries:**
   ```bash
   git status --short | awk '{print $2}' | grep -E '^\.swarm/$|^ruvector\.db$|^0$|^\.claude/homunculus/'
   ```
   Report which strays are present in this working tree.

3. **Confirm none of them are staged.** The `git status --short`
   output uses two columns: index status (col 1) and worktree
   status (col 2). A line like `A  ruvector.db` or `M  .swarm/x`
   (status letter in col 1, not space) means the stray got staged.
   If any stray is staged, **stop** and ask the user to `git
   restore --staged` it before continuing.

4. **Offer to update `.gitignore`** with the three file-shaped
   strays (`/0`, `ruvector.db`, `.swarm/`). The fourth (homunculus
   observations) is already covered by the broader `.claude/` rule;
   it shows up only because it was tracked before the rule was
   added. Don't touch tracking unless the user explicitly asks
   `git rm --cached`.

   Suggested `.gitignore` addition (append, do not replace):
   ```
   # Session/tool runtime state — not part of FuFirE
   /0
   ruvector.db
   .swarm/
   ```

5. **Print a green-light summary** when the working tree is safe
   to ship:
   ```
   Staging clean — N feature files staged, M strays left in
   working tree (untracked). Hand off to /ship.
   ```

### Guardrails

- This skill is a **pre-ship guard**, not a stager. It must not
  call `git add` on feature files automatically — the user (or
  /ship) decides what to stage.
- If a stray is already tracked (e.g. `.claude/homunculus/`),
  recommend the safe path (leave it modified in the working tree)
  but **never** propose `git rm --cached` without explicit user
  approval — that's a real removal from the index that affects
  future sessions.
- Do NOT modify `.gitignore` without asking. Show the diff
  ("we'd append these 3 lines"), then act only on confirmation.
- If the user is on a branch that already has the four entries
  in `.gitignore`, skip the offer and just print the green light.
