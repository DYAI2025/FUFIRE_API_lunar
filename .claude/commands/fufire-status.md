## Context

The FuFirE BaZi-Precision-V2 initiative tracks progress in three
canonical files:

- `3-code/tasks.md` — `Phase BAZI-PRECISION-V2` table with FBP-XX-XXX
  task rows (Status column: `Done` / `In Progress` / `Todo` / `Blocked`).
- `docs/precision/deviations.md` — DEV-YYYY-NNN register with
  severity (P0/P1/P2/P3) and status (Open / Mitigated / Resolved / …).
- `docs/release/bazi_precision_release_gate.md` — phase stop-gates
  (§1) and default-switch criteria (§2).

Users typically ask "wie weit sind wir?" or "where are we?" — they
expect a synthesis of these three files, not the raw contents.

## Your Task

Produce a compact phase-progress dashboard.

### Steps

1. **Count FBP tasks per phase** by parsing `3-code/tasks.md`.
   ```bash
   awk '/^\| FBP-[0-9]+-[0-9]+/ {match($0, /FBP-[0-9]+/); phase=substr($0,RSTART,RLENGTH); split($0, c, "|"); status=c[5]; gsub(/^ +| +$/, "", status); print phase, status}' 3-code/tasks.md \
     | sort | uniq -c | sort -k2
   ```
   Group by phase prefix (`FBP-00`, `FBP-01`, … `FBP-07`). Compute
   Done / In Progress / Todo per phase plus a grand total.

2. **List open P0/P1 deviations** from `docs/precision/deviations.md`.
   Each deviation block starts with `## DEV-YYYY-NNN — <title>` and
   has a `Severity:` and `Status:` line. Show only entries whose
   Status is not `Resolved` and severity is `P0` or `P1`.

3. **Evaluate the stop-gate for the NEXT phase.** Read the
   `Phase N → Phase N+1` section in
   `docs/release/bazi_precision_release_gate.md`, compare each
   bullet against what's actually Done in tasks.md / present on
   disk (e.g. `bazi_engine/time_context.py` for Phase 1 → Phase 2).
   Report `✅`, `❌`, or `🟡 pending`.

4. **Output a table + a stop-gate verdict + a "next sensible
   moves" menu**. Format:

   ```
   ## FuFirE BAZI-PRECISION-V2 — Status

   | Phase | Done | In Progress | Todo | Notes |
   |---|---|---|---|---|
   | 0 — Baseline & Governance | 8 | 0 | 0 | shipped 1faff5e |
   | 1 — Effective Time Model  | 5 | 0 | 0 | shipped bfd3486 |
   | 2 — BaZi Core             | 0 | 0 | 8 | next |
   | …                                                |

   ### Open deviations (P0/P1)
   - DEV-2026-001 (P0): …
   - DEV-2026-004 (P0): blocker for v2 default-switch
   - …

   ### Stop-gate Phase X → Phase X+1
   | Gate | Status |
   |---|---|
   | … | ✅ / ❌ / 🟡 |

   ### Next sensible moves
   | Option | Effort | Effect |
   |---|---|---|
   | A — … | … | … |
   | B — … | … | … |
   ```

5. **Language**: if the user's invocation message (or recent
   conversation) is German, respond in German; otherwise English.
   FuFirE users frequently switch.

### Guardrails

- Do NOT modify tasks.md, deviations.md, or the release gate doc.
  This skill is read-only.
- Do NOT invent task IDs. If a file is missing or malformed, report
  the missing file explicitly rather than fabricating data.
- Do NOT confuse `Mitigated (v1)` with `Resolved` — deviations that
  are mitigated still need follow-up (track them separately).
- Show recent commit hashes only if the user asked for them or if
  the phase is explicitly marked shipped in tasks.md notes.
